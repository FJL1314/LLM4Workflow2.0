## Your Task
Generate 2-4 **additional** evaluation dimensions that:
1. **Complement** the generic rubric (not duplicate it)
2. Are **unique** to this specific task or domain
3. Are **atomic** - each dimension should be independent and non-overlapping with others
4. Focus on **task-specific nuances** that the generic rubric doesn't capture

### Examples of Good Task-Specific Dimensions:
- **Audio-Visual Synchronization Quality** (for video editing tasks)
- **Multi-language Translation Accuracy** (for translation tasks)
- **Real-time Processing Latency** (for streaming tasks)
- **Privacy and Security Compliance** (for data handling tasks)

### Anti-Patterns to Avoid:
Don't use "Tool Selection" (overlaps with Semantic Tool Alignment)
Don't use "DAG Structure" (overlaps with Structural Integrity)
Don't use "Efficiency" (overlaps with Resource Optimization)
Don't create overlapping dimensions (e.g., both "Audio Quality" and "Sound Clarity")

For each dimension:
- **Theme**: Unique, descriptive name (1-5 words)
- **Description**: What this dimension measures and why it's critical for THIS task
- **Tips**: 3-5 specific, actionable evaluation criteria
- **Atomicity**: Ensure each tip relates to ONE aspect (don't mix multiple concerns)
